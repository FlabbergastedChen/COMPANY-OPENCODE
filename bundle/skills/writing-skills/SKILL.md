# 编写 Skill 文档

## 目标
产出可执行、可复用、低歧义的 `SKILL.md`。

## 基本结构
1. frontmatter（name/description）
2. 作用
3. 执行步骤
4. 输出要求
5. 护栏（可选）

## 编写原则
- 任务导向：描述做什么与何时使用
- 过程可执行：步骤可直接操作
- 输出可验收：结果标准清晰
- 约束明确：避免误用与越界

## 质量检查
- 命名与目录一致
- 描述不空泛
- 步骤不互相冲突
- 输出要求可验证
